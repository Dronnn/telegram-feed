# Plan v3: Баги после первого запуска

## Цель
Исправить баги, обнаруженные при первом реальном использовании приложения после успешной авторизации.

## Баги

### Bug 1: Настройки — пустой список каналов без пояснения
**Симптом:** Пользователь залогинился в аккаунт, в котором нет подписок на каналы. Открывает Settings → секция Channels пустая, ничего не показывается.
**Ожидание:** Показывать сообщение "You have no channels" (или аналогичное empty state) вместо пустоты.

### Bug 2: Настройки — новые каналы не появляются без перезапуска
**Симптом:** Пользователь подписался на новый канал в Telegram, но в Settings приложения канал не появляется. Приходится полностью закрывать и открывать приложение.
**Ожидание:** В Settings → Channels добавить pull-to-refresh для повторного запроса списка каналов из TDLib.

### Bug 3: Лента — обновление после изменения выбранных каналов
**Симптом:** Пользователь включил/отключил каналы в Settings, закрыл Settings, вернулся на FeedView — контент не обновляется, показывается ошибка. Только после перезапуска приложения лента загружается.
**Ожидание:**
- После закрытия Settings лента автоматически обновляется.
- Если канал отключён — его сообщения пропадают из ленты, но scroll position НЕ меняется. Пользователь остаётся на том же месте (видит то же сообщение в центре экрана). Если текущее сообщение было из отключённого канала — показывается следующее по списку.
- Если канал включён — его сообщения добавляются в ленту, scroll position НЕ меняется.
- Никакого скролла вверх/вниз при обновлении ленты.

### Bug 4: Ссылка на пост в конце карточки не работает
**Симптом:** В конце каждой карточки сообщения приложение добавляет ссылку на оригинальный пост (например, "rozetka/44777"). При нажатии на эту ссылку ничего не происходит. Ссылки, добавленные самим каналом внутри текста сообщения, работают нормально (открывают Telegram) — это ОК, пусть так и остаётся.
**Ожидание:** При нажатии на ссылку поста (добавленную приложением) — открывается ChannelSheetView снизу с лентой этого канала, проскроленной к конкретному сообщению. Не перебрасывать в Telegram.

### Bug 5: ChannelSheet — не проскроллен к нужному сообщению + показывает только 1 сообщение
**Симптом:** При нажатии на название канала в шапке карточки открывается ChannelSheetView, но:
- Лента не проскроллена к сообщению, с которого пользователь перешёл
- Показывается только 1 сообщение вместо полной ленты канала
**Ожидание:**
- Показывать все сообщения канала (старые сверху, новые снизу)
- Проскроллить к сообщению, с которого пользователь перешёл
- Бесконечный скролл вверх (подгрузка старых) и вниз (новые)

### Bug 6: Лента — загружается только 1 сообщение вместо 30
**Симптом:** После успешной загрузки ленты отображается только 1 сообщение.
**Ожидание:**
- При первом входе (нет сохранённой позиции) загружать последние 30 сообщений для каждого выбранного канала
- Показывать ленту с самого нового сообщения (пользователь внизу)
- При скролле вверх — бесконечная подгрузка старых сообщений (по порциям)
- Вниз — все новые сообщения отображаются в реальном времени

### Bug 7: Лента — новые сообщения не появляются в реальном времени
**Симптом:** Пользователь читает ленту. В это время канал публикует новое сообщение. Сообщение не появляется в ленте автоматически — нужно делать pull-to-refresh или перезапускать приложение.
**Ожидание:** Новые сообщения из выбранных каналов должны появляться в ленте автоматически в реальном времени. Приложение подписано на обновления TDLib (`updateNewMessage`) и добавляет новые сообщения в конец ленты без каких-либо действий пользователя. Pull-to-refresh — это ручное действие «на всякий случай», а не основной способ получения новых сообщений.
